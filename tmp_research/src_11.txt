<h6 class="ltx_title ltx_title_abstract">Abstract</h6>
<p class="ltx_p" id="2.1">We introduce <span class="ltx_text ltx_font_smallcaps" id="2.1.1">Self-Discover</span>, a general framework for LLMs to self-discover the task-intrinsic reasoning structures to tackle complex reasoning problems that are challenging for typical prompting methods.
Core to the framework is a self-discovery process where LLMs select multiple atomic reasoning modules such as critical thinking and step-by-step thinking, and compose them into an explicit reasoning structure for LLMs to follow during decoding.
<span class="ltx_text ltx_font_smallcaps" id="2.1.2">Self-Discover</span> substantially improves GPT-4 and PaLM 2’s performance on challenging reasoning benchmarks such as BigBench-Hard, grounded agent reasoning, and MATH, by as much as 32% compared to Chain of Thought (CoT).
Furthermore, <span class="ltx_text ltx_font_smallcaps" id="2.1.3">Self-Discover</span> outperforms inference-intensive methods such as CoT-Self-Consistency by more than 20%, while requiring 10-40x fewer inference compute.
Finally, we show that the self-discovered reasoning structures are universally applicable across model families: from PaLM 2-L to GPT-4, and from GPT-4 to Llama2, and share commonalities with human reasoning patterns.</p>

<section class="ltx_section" id="S1">
<h2 class="ltx_title ltx_title_section">
<span class="ltx_tag ltx_tag_section">1 </span>Introduction</h2>
<div class="ltx_para" id="S1.p1">
<p class="ltx_p" id="S1.p1.1">Large Language Models (LLM) <cite class="ltx_cite ltx_citemacro_citep">(Brown et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib3" title="">2020</a>; Chowdhery et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib6" title="">2022</a>; OpenAI, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib28" title="">2023b</a>; Anil et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib1" title="">2023</a>)</cite> powered by
transformers <cite class="ltx_cite ltx_citemacro_citep">(Vaswani et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib37" title="">2017</a>)</cite> have produced impressive breakthroughs in generating coherent texts <cite class="ltx_cite ltx_citemacro_citep">(OpenAI, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib26" title="">2022</a>)</cite>, and following instructions <cite class="ltx_cite ltx_citemacro_citep">(Zhong et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib47" title="">2021</a>; Mishra et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib23" title="">2022c</a>; Wei et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib40" title="">2021</a>; Chung et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib7" title="">2022</a>; Ouyang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib29" title="">2022</a>)</cite>.
In pursuit of the goal to enhance LLMs’ capability to <span class="ltx_text ltx_font_italic" id="S1.p1.1.1">reason</span> and solve complex problems, various prompting methods have been proposed, drawing inspirations from cognitive theories of how humans reason. For example, few-shot and zero-shot chain-of-thought (CoT) <cite class="ltx_cite ltx_citemacro_citep">(Nye et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib25" title="">2021</a>; Wei et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib41" title="">2022</a>; Kojima et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib17" title="">2022</a>; Yasunaga et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib45" title="">2023</a>)</cite> resembles how humans solve problems step-by-step, decomposition-based prompting <cite class="ltx_cite ltx_citemacro_citep">(Zhou et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib48" title="">2022a</a>; Drozdov et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib9" title="">2022</a>; Patel et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib30" title="">2022</a>; Hao et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib13" title="">2023</a>; Khot et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib16" title="">2022</a>)</cite> is inspired by how humans breakdown a complex problem into a series of smaller subproblems, and then solve those subproblems one by one <cite class="ltx_cite ltx_citemacro_citep">(Polya, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib31" title="">2004</a>)</cite>, and step-back prompting <cite class="ltx_cite ltx_citemacro_citep">(Zheng et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib46" title="">2023</a>)</cite> is motivated by how humans reflect on task nature to derive general principles.
However, a fundamental limitation is that each technique itself serves as an atomic reasoning module making an implicit prior assumption of the process on how to tackle a given task. Instead, we argue that each task has a unique intrinsic structure underlying the reasoning process involved in solving it efficiently. For instance, least-to-most prompting <cite class="ltx_cite ltx_citemacro_citep">(Zhou et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib48" title="">2022a</a>; Drozdov et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib9" title="">2022</a>)</cite> has shown to be much more effective than CoT <cite class="ltx_cite ltx_citemacro_citep">(Wei et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib41" title="">2022</a>)</cite> at solving tasks such as symbolic manipulation and compositional generalization, due to the decomposition structure of the tasks.</p>
</div>
<div class="ltx_para" id="S1.p2">
<p class="ltx_p" id="S1.p2.1">This paper aims at self-discovering the underlying reasoning structure unique to each task, while being highly efficient in terms of computation.
Our approach, <span class="ltx_text ltx_font_smallcaps" id="S1.p2.1.1">Self-Discover</span>, is inspired by how humans internally devise a reasoning <span class="ltx_text ltx_font_italic" id="S1.p2.1.2">program</span> for problem-solving <cite class="ltx_cite ltx_citemacro_citep">(Newell et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib24" title="">1958</a>; Rasmussen, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib32" title="">1983</a>)</cite>, as illustrated in Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S2.F2" title="Figure 2 ‣ 2 Self-Discovering Reasoning Structures for Problem-Solving ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">2</span></a> .
From a set of atomic reasoning modules described in natural language such as “<span class="ltx_text ltx_font_italic" id="S1.p2.1.3">breakdown into sub tasks</span>” and “<span class="ltx_text ltx_font_italic" id="S1.p2.1.4">critical thinking</span>”, an LLM, and task examples without labels,
<span class="ltx_text ltx_font_smallcaps" id="S1.p2.1.5">Self-Discover</span> composes a coherent reasoning structure intrinsic to the task (Stage 1) and then solves instances of the task using the discovered structure (Stage 2).
Stage 1 operates at the task-level and uses three actions to guide the LLM to generate a reasoning structure for the task.
At Stage 2, during the final decoding, the LLM simply follows the self-discovered structure to arrive at the final answer.</p>
</div>
<div class="ltx_para" id="S1.p3">
<p class="ltx_p" id="S1.p3.1">Solving problems using <span class="ltx_text ltx_font_smallcaps" id="S1.p3.1.1">Self-Discover</span> brings several benefits compared to other methods for LLM reasoning.
First, the discovered reasoning structure is grounded in atomic reasoning modules benefiting from the strengths of <span class="ltx_text ltx_font_italic" id="S1.p3.1.2">multiple</span> reasoning modules in contrast to applying a priori module such as CoT.
Second, <span class="ltx_text ltx_font_smallcaps" id="S1.p3.1.3">Self-Discover</span> is efficient in computation as it only requires 3 more inference steps on the <span class="ltx_text ltx_font_italic" id="S1.p3.1.4">task-level</span>,
while being more performant than inference-heavy ensemble approaches such as self-consistency <cite class="ltx_cite ltx_citemacro_citep">(Wang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib39" title="">2022</a>)</cite>.
Lastly, the discovered reasoning structure is intrinsic to the task, and conveys LLMs’ insights about the task in a more <span class="ltx_text ltx_font_italic" id="S1.p3.1.5">interpretable</span> way than the optimized prompts <cite class="ltx_cite ltx_citemacro_citep">(Zhou et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib50" title="">2022b</a>; Yang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib42" title="">2023</a>)</cite>.
</p>
</div>
<div class="ltx_para" id="S1.p4">
<p class="ltx_p" id="S1.p4.1">We test <span class="ltx_text ltx_font_smallcaps" id="S1.p4.1.1">Self-Discover</span> on 25 challenging reasoning tasks including Big Bench-Hard (BBH) <cite class="ltx_cite ltx_citemacro_citep">(Suzgun et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib35" title="">2022</a>)</cite>, Thinking for Doing (T4D) <cite class="ltx_cite ltx_citemacro_citep">(Zhou et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib49" title="">2023</a>)</cite> and MATH <cite class="ltx_cite ltx_citemacro_citep">(Hendrycks et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib15" title="">2021</a>)</cite>.
<span class="ltx_text ltx_font_smallcaps" id="S1.p4.1.2">Self-Discover</span> outperforms CoT on 21/25 task with performance gains up to 42% (Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S0.F1" title="Figure 1 ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">1</span></a>), highlighting the advantage of the self-discovered reasoning structure composed from the atomic reasoning modules against a single a priori CoT module. Furthermore, we demonstrate that <span class="ltx_text ltx_font_smallcaps" id="S1.p4.1.3">Self-Discover</span> achieves superior performance against inference-heavy methods such as CoT + Self-Consistency and majority voting of every module while requiring 10-40x fewer inference compute (Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S4.F5" title="Figure 5 ‣ 4.2 Which Types of Problems Do Self-Discover Help the Most? ‣ 4 Results ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">5</span></a>).
Finally, we compare <span class="ltx_text ltx_font_smallcaps" id="S1.p4.1.4">Self-Discover</span> with prompts optimized (OPRO) using a training set <cite class="ltx_cite ltx_citemacro_citep">(Yang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib42" title="">2023</a>)</cite> (Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S5.F9" title="Figure 9 ‣ Applying PaLM 2-L Discovered Structures to GPT-4 ‣ 5.2 Towards Universality of Discovered Reasoning Structures ‣ 5 Deep Diving Into Self-Discovered Reasoning Structures ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">9</span></a>). We find that <span class="ltx_text ltx_font_smallcaps" id="S1.p4.1.5">Self-Discover</span> still performs on par or better than OPRO while the self-discovered reasoning structure are much more interpretable.
</p>
</div>
<div class="ltx_para" id="S1.p5">
<p class="ltx_p" id="S1.p5.1">We conduct a set of analysis to understand the effectiveness of <span class="ltx_text ltx_font_smallcaps" id="S1.p5.1.1">Self-Discover</span>.
By breaking down BBH tasks into 4 different categories, we find that <span class="ltx_text ltx_font_smallcaps" id="S1.p5.1.2">Self-Discover</span> performs best on tasks requiring world knowledge and has a moderate performance boost on algorithmic tasks compared to CoT (Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S4.F4" title="Figure 4 ‣ 4.2 Which Types of Problems Do Self-Discover Help the Most? ‣ 4 Results ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">4</span></a>). This is further confirmed by the error analysis on MATH, where 74.7% model failures comes from computation errors (e.g. math). We also take a closer look at the self-discovered reasoning structures, and show the universality of them by transferability study from PaLM 2-L to GPT-4, and from GPT-4 to Llama-2-70B. We hope to encourage more future work on structured reasoning for solving challenging problems using LLMs.</p>
</div>
</section>
<section class="ltx_section" id="S2">
<h2 class="ltx_title ltx_title_section">
<span class="ltx_tag ltx_tag_section">2 </span>Self-Discovering Reasoning Structures for Problem-Solving</h2>
<figure class="ltx_figure" id="S2.F2"><img alt="Refer to caption" class="ltx_graphics ltx_centering ltx_img_landscape" height="263" id="S2.F2.g1" src="x2.png" width="747"/>
<figcaption class="ltx_caption ltx_centering"><span class="ltx_tag ltx_tag_figure"><span class="ltx_text" id="S2.F2.3.1.1" style="font-size:90%;">Figure 2</span>: </span><span class="ltx_text ltx_font_bold" id="S2.F2.4.2" style="font-size:90%;">Illustration of using <span class="ltx_text ltx_font_smallcaps" id="S2.F2.4.2.1">Self-Discover</span> for problem-solving<span class="ltx_text ltx_font_medium" id="S2.F2.4.2.2">. Given a generative LM, task, and seed reasoning module descriptions, we guide LMs to generate a reasoning structure in <span class="ltx_text ltx_font_italic" id="S2.F2.4.2.2.1">key-value</span> format to solve the task. Finally, models can follow the self-discovered structures to solve the every instance from the task by filling in the values in JSON step-by-step.</span></span></figcaption>
</figure>
<div class="ltx_para" id="S2.p1">
<p class="ltx_p" id="S2.p1.1">We take inspiration from how humans use prior knowledge and skills to devise a reasoning program to solve problems <cite class="ltx_cite ltx_citemacro_citep">(Newell et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib24" title="">1958</a>; Rasmussen, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib32" title="">1983</a>)</cite>.
When we face a new problem, we often first search internally what knowledge and skills from our prior experience might be helpful to solve it.
Then we will attempt to apply relevant knowledge and skills to this task.
And finally we will connect multiple individual skills and knowledge to solve the problem.
We design <span class="ltx_text ltx_font_smallcaps" id="S2.p1.1.1">Self-Discover</span> to enact these steps into two stages as illustrated in Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S2.F2" title="Figure 2 ‣ 2 Self-Discovering Reasoning Structures for Problem-Solving ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">2</span></a>.</p>
</div>
<div class="ltx_para" id="S2.p2">
<p class="ltx_p" id="S2.p2.1">Given a task and a set of reasoning module descriptions representing high-level problem-solving heuristics such as “<span class="ltx_text ltx_font_italic" id="S2.p2.1.1">Use critical thinking</span>” and “<span class="ltx_text ltx_font_italic" id="S2.p2.1.2">Let’s think step by step</span>”, Stage 1 of <span class="ltx_text ltx_font_smallcaps" id="S2.p2.1.3">Self-Discover</span> aims to uncover the intrinsic reasoning structure for solving this task via meta-reasoning.
Specifically, we uses three meta-prompts to guide LLMs to select, adapt, and implement an actionable reasoning structure with no labels or training required.
We format the structure in key-value pairs similar to JSON due to interpretability and findings on following JSON boosts reasoning and generation quality <cite class="ltx_cite ltx_citemacro_citep">(Zhou et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib49" title="">2023</a>; OpenAI, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib27" title="">2023a</a>)</cite>.
The structure of the meta-prompts and full prompts are shown in Appendix.
Stage 1 operates on <span class="ltx_text ltx_font_italic" id="S2.p2.1.4">task-level</span>, meaning we only need to run <span class="ltx_text ltx_font_smallcaps" id="S2.p2.1.5">Self-Discover</span> once for each task.
Then, in Stage 2, we can simply use the discovered reasoning structure to solve every <span class="ltx_text ltx_font_italic" id="S2.p2.1.6">instance</span> of the given task by instructing models to follow the provided structure by filling each key and arrive at a final answer.
</p>
</div>
<section class="ltx_subsection" id="S2.SS1">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">2.1 </span>Stage 1: Self-Discover Task-Specific Structures</h3>
<div class="ltx_para" id="S2.SS1.p1">
<p class="ltx_p" id="S2.SS1.p1.1">The first stage consists of three actions: 1) SELECT, where relevant reasoning modules for task-solving are chosen from the set of reasoning module descriptions; 2) ADAPT, where descriptions of selected reasoning modules are rephrased to be more specific to the task at hand; and 3) IMPLEMENT, where the adapted reasoning descriptions are implemented into a structured actionable plan so that the task can be solved by following the structure.</p>
</div>
<figure class="ltx_figure" id="S2.F3"><img alt="Refer to caption" class="ltx_graphics ltx_centering ltx_img_landscape" height="320" id="S2.F3.g1" src="x3.png" width="747"/>
<figcaption class="ltx_caption ltx_centering"><span class="ltx_tag ltx_tag_figure"><span class="ltx_text" id="S2.F3.3.1.1" style="font-size:90%;">Figure 3</span>: </span><span class="ltx_text ltx_font_bold" id="S2.F3.4.2" style="font-size:90%;">Illustration of three actions of <span class="ltx_text ltx_font_smallcaps" id="S2.F3.4.2.1">Self-Discover</span><span class="ltx_text ltx_font_medium" id="S2.F3.4.2.2">. We use LMs to compose a coherent reasoning structure by selecting relevant modules, adapting to task-specific descriptions, and implement a reasoning structure in JSON.</span></span></figcaption>
</figure>

For example, “<span class="ltx_text ltx_font_italic" id="S2.SS1.SSS0.Px1.p1.5.2">reflective thinking</span>” might help search for first-principle theories on science problems, while “<span class="ltx_text ltx_font_italic" id="S2.SS1.SSS0.Px1.p1.5.3">creative thinking</span>” helps on generating a novel continuation to a story.

<section class="ltx_section" id="S3">
<h2 class="ltx_title ltx_title_section">
<span class="ltx_tag ltx_tag_section">3 </span>Experiment Setup</h2>
<section class="ltx_subsection" id="S3.SS1">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">3.1 </span>Tasks</h3>
<div class="ltx_para" id="S3.SS1.p1">
<p class="ltx_p" id="S3.SS1.p1.1">We focus on diverse reasoning benchmarks that are still challenging for LLMs: BIG-Bench Hard (BBH) <cite class="ltx_cite ltx_citemacro_citep">(Suzgun et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib35" title="">2022</a>)</cite> contains 23 carefully-selected challenging tasks from BIG-Bench <cite class="ltx_cite ltx_citemacro_citep">(Srivastava et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib34" title="">2023</a>)</cite>.
BBH tasks cover a diverse range of reasoning problems spanning the following 4 categories according to their authors: 1) Algorithmic and Multi-Step Arithmetic Reasoning, 2) Natural Language Understanding, 3) Use of World Knowledge, and 4) Multilingual Knowledge and Reasoning. We also test on a grounded social agent reasoning task called Thinking for Doing (T4D) where models must leverage mental state reasoning to determine actions to perform <cite class="ltx_cite ltx_citemacro_citep">(Zhou et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib49" title="">2023</a>)</cite>, where GPT-4 with CoT only reaches around 50%. Finally, we subsample 200 examples from the MATH <cite class="ltx_cite ltx_citemacro_citep">(Hendrycks et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib15" title="">2021</a>)</cite> test set, and generate instance-level reasoning structures via a one-shot demonstration to adapt to the complexity of MATH tasks. For evaluations, we use accuracy to measure the model performance on BBH, T4D and MATH (details can be found in Appendix <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#A2" title="Appendix B Evaluation Details ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">B</span></a>).</p>
</div>
</section>
<section class="ltx_subsection" id="S3.SS2">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">3.2 </span>Models</h3>
<div class="ltx_para" id="S3.SS2.p1">
<p class="ltx_p" id="S3.SS2.p1.1">We use several state-of-the-art LLMs: GPT-4 (gpt-4-turbo-preview) <cite class="ltx_cite ltx_citemacro_citep">(OpenAI, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib28" title="">2023b</a>)</cite>, GPT-3.5-turbo (ChatGPT) <cite class="ltx_cite ltx_citemacro_citep">(OpenAI, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib26" title="">2022</a>)</cite><span class="ltx_note ltx_role_footnote" id="footnote1"><sup class="ltx_note_mark">1</sup><span class="ltx_note_outer"><span class="ltx_note_content"><sup class="ltx_note_mark">1</sup><span class="ltx_tag ltx_tag_note">1</span>accessed in October-December 2023</span></span></span>, instruction-tuned PaLM 2-L <cite class="ltx_cite ltx_citemacro_citep">(Anil et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib1" title="">2023</a>)</cite><span class="ltx_note ltx_role_footnote" id="footnote2"><sup class="ltx_note_mark">2</sup><span class="ltx_note_outer"><span class="ltx_note_content"><sup class="ltx_note_mark">2</sup><span class="ltx_tag ltx_tag_note">2</span>For MATH, we use a PaLM 2-L model with a stronger instruction tuning to enable better instruction following of more complex reasoning structures.</span></span></span>, and an open-source LLM Llama2-70B <cite class="ltx_cite ltx_citemacro_citep">(Touvron et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib36" title="">2023</a>)</cite>.</p>
</div>
</section>
<section class="ltx_subsection" id="S3.SS3">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">3.3 </span>Baselines</h3>
<div class="ltx_para" id="S3.SS3.p1">
<p class="ltx_p" id="S3.SS3.p1.1">We compare <span class="ltx_text ltx_font_smallcaps" id="S3.SS3.p1.1.1">Self-Discover</span> with other zero-shot prompting methods for LLM reasoning:</p>
<ul class="ltx_itemize" id="S3.I1">
<li class="ltx_item" id="S3.I1.i1" style="list-style-type:none;">
<span class="ltx_tag ltx_tag_item">•</span>
<div class="ltx_para" id="S3.I1.i1.p1">
<p class="ltx_p" id="S3.I1.i1.p1.1"><span class="ltx_text ltx_font_bold" id="S3.I1.i1.p1.1.1">Direct Prompting</span>, where model directly generates the answer without intermediate reasoning steps.</p>
</div>
</li>
<li class="ltx_item" id="S3.I1.i2" style="list-style-type:none;">
<span class="ltx_tag ltx_tag_item">•</span>
<div class="ltx_para" id="S3.I1.i2.p1">
<p class="ltx_p" id="S3.I1.i2.p1.1"><span class="ltx_text ltx_font_bold" id="S3.I1.i2.p1.1.1">CoT</span> <cite class="ltx_cite ltx_citemacro_citep">(Wei et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib41" title="">2022</a>; Kojima et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib17" title="">2022</a>)</cite>, where models are prompted to generate a reasoning process leading to the final answer.</p>
</div>
</li>
<li class="ltx_item" id="S3.I1.i3" style="list-style-type:none;">
<span class="ltx_tag ltx_tag_item">•</span>
<div class="ltx_para" id="S3.I1.i3.p1">
<p class="ltx_p" id="S3.I1.i3.p1.1"><span class="ltx_text ltx_font_bold" id="S3.I1.i3.p1.1.1">Plan-and-Solve</span> <cite class="ltx_cite ltx_citemacro_citep">(Wang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib38" title="">2023</a>)</cite>, where models are prompted to first generate a plan and then solve the problem. <span class="ltx_text ltx_font_smallcaps" id="S3.I1.i3.p1.1.2">Self-Discover</span> differs by grounding the reasoning structure in atomic reasoning modules, and prompting the decoding to follow the explicit key-value reasoning structure.</p>
</div>
</li>
</ul>
</div>
<div class="ltx_para" id="S3.SS3.p2">
<p class="ltx_p" id="S3.SS3.p2.1">Next, we also consider other baselines that make use of the raw seed reasoning modules (RM) we pass to <span class="ltx_text ltx_font_smallcaps" id="S3.SS3.p2.1.1">Self-Discover</span>. We compare with the following methods’ performance and the inference call efficiency on a subset of tasks.</p>
<ul class="ltx_itemize" id="S3.I2">
<li class="ltx_item" id="S3.I2.i1" style="list-style-type:none;">
<span class="ltx_tag ltx_tag_item">•</span>
<div class="ltx_para" id="S3.I2.i1.p1">
<p class="ltx_p" id="S3.I2.i1.p1.1"><span class="ltx_text ltx_font_bold" id="S3.I2.i1.p1.1.1">CoT-Self-Consistency</span> <cite class="ltx_cite ltx_citemacro_citep">(Wang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib39" title="">2022</a>)</cite>, we sample multiple outputs from LLM with CoT and aggregate answers to get the final answer. We compare this method on a subset of tasks due to the cost of repetitive queries.
</p>
</div>
</li>
<li class="ltx_item" id="S3.I2.i2" style="list-style-type:none;">
<span class="ltx_tag ltx_tag_item">•</span>
<div class="ltx_para" id="S3.I2.i2.p1">
<p class="ltx_p" id="S3.I2.i2.p1.1"><span class="ltx_text ltx_font_bold" id="S3.I2.i2.p1.1.1">Majority voting of each RM</span>: we prompt models to solve the tasks by appending each RM and use majority voting of all answers to get the final answer. We examine whether integrating <span class="ltx_text ltx_font_italic" id="S3.I2.i2.p1.1.2">multiple</span> RMs into a coherent reasoning structure is advantageous to applying each RM to solve the task and use majority voting to ensemble them post-hoc, which costs much more inference computation.</p>
</div>
</li>
<li class="ltx_item" id="S3.I2.i3" style="list-style-type:none;">
<span class="ltx_tag ltx_tag_item">•</span>
<div class="ltx_para" id="S3.I2.i3.p1">
<p class="ltx_p" id="S3.I2.i3.p1.1"><span class="ltx_text ltx_font_bold" id="S3.I2.i3.p1.1.1">Best of each RM</span>: this method assumes that we have access to oracle labels and uses the highest accuracy from applying each RM. We compare with this to examine whether <span class="ltx_text ltx_font_smallcaps" id="S3.I2.i3.p1.1.2">Self-Discover</span> competes with methods that depend on perfect prior knowledge of which RM to use on a new task.</p>
</div>
</li>
</ul>
</div>
<div class="ltx_para" id="S3.SS3.p3">
<p class="ltx_p" id="S3.SS3.p3.1">Furthermore, for analysis on universality of reasoning structures, we compare with a prompt-optimization method that require a <span class="ltx_text ltx_font_italic" id="S3.SS3.p3.1.1">training</span> set to improve prompts: LLMs as optimizers (<span class="ltx_text ltx_font_bold" id="S3.SS3.p3.1.2">OPRO</span>) <cite class="ltx_cite ltx_citemacro_citep">(Yang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib42" title="">2023</a>)</cite>.
We aim to show that when we apply structures or prompts optimized from one model, the reasoning structures can retain more performance gains than the wordings of prompts.</p>
</div>
</section>
</section>

<span class="ltx_tag ltx_tag_section">4 </span>Results</h2>
<div class="ltx_para" id="S4.p1">
<p class="ltx_p" id="S4.p1.1">We answer the following questions through experimental results: 1) <span class="ltx_text ltx_font_italic" id="S4.p1.1.1">Does discovering reasoning structures improve LLM reasoning capabilities?</span> (<a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S4.SS1" title="4.1 Does Self-Discover Improve LLM Reasoning? ‣ 4 Results ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">4.1</span></a>) 2) <span class="ltx_text ltx_font_italic" id="S4.p1.1.2">Which categories of problems do <span class="ltx_text ltx_font_smallcaps" id="S4.p1.1.2.1">Self-Discover</span> perform the best?</span> (<a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S4.SS2" title="4.2 Which Types of Problems Do Self-Discover Help the Most? ‣ 4 Results ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">4.2</span></a>) and 3) <span class="ltx_text ltx_font_italic" id="S4.p1.1.3">Can <span class="ltx_text ltx_font_smallcaps" id="S4.p1.1.3.1">Self-Discover</span> boost LLM performance efficiently?</span> (<a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S4.SS3" title="4.3 How Efficient is Self-Discover? ‣ 4 Results ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">4.3</span></a>)

<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">4.1 </span>Does <span class="ltx_text ltx_font_smallcaps" id="S4.SS1.1.1">Self-Discover</span> Improve LLM Reasoning?</h3>
<div class="ltx_para" id="S4.SS1.p1">
<p class="ltx_p" id="S4.SS1.p1.1"><span class="ltx_text ltx_font_bold" id="S4.SS1.p1.1.1">Overall, <span class="ltx_text ltx_font_smallcaps" id="S4.SS1.p1.1.1.1">Self-Discover</span> improves PaLM 2-L and GPT-4’s reasoning across diverse set of reasoning tasks.</span>
Table <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S4.T1" title="Table 1 ‣ 4 Results ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">1</span></a> shows the overall results on complex reasoning tasks of BBH, T4D and MATH using PaLM 2-L and GPT-4.

Similar gains (6% and 8%) are observed when <span class="ltx_text ltx_font_smallcaps" id="S4.SS1.p2.1.2">Self-Discover</span> is applied to GPT-4. Breakdown results of each task’s improvement over direct answering and CoT of PaLM 2-L are shown in Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S0.F1" title="Figure 1 ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">1</span></a>, where we find <span class="ltx_text ltx_font_smallcaps" id="S4.SS1.p2.1.3">Self-Discover</span> outperforms them on over 20/24 tasks. For a per-task performance for all 23 BBH tasks, please refer to Appendix <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#A3" title="Appendix C BBH Per Task Performance ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">C</span></a>.

<section class="ltx_subsection" id="S4.SS3">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">4.3 </span>How Efficient is <span class="ltx_text ltx_font_smallcaps" id="S4.SS3.1.1">Self-Discover</span>?</h3>
<div class="ltx_para" id="S4.SS3.p1">
<p class="ltx_p" id="S4.SS3.p1.1"><span class="ltx_text ltx_font_bold ltx_font_smallcaps" id="S4.SS3.p1.1.1">Self-Discover<span class="ltx_text ltx_font_upright" id="S4.SS3.p1.1.1.1"> achieves better performance while requiring 10-40x fewer inference computer compared to self-consistency or majority voting.</span></span>
Here we examine a subset of 2 tasks from BBH and present a more thorough comparison of methods including those requiring many inference calls that are too costly to run on all 24 tasks.
Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S4.F5" title="Figure 5 ‣ 4.2 Which Types of Problems Do Self-Discover Help the Most? ‣ 4 Results ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">5</span></a> shows average accuracy and number of inference calls required per instance for each method using GPT-4.
<span class="ltx_text ltx_font_bold" id="S4.SS3.p1.1.2">Accuracy wise (y-axis)</span>, we find that <span class="ltx_text ltx_font_smallcaps" id="S4.SS3.p1.1.3">Self-Discover</span> outperforms other baselines even those that require repeated inference calls such as CoT-self-consistency and majority voting of applying each RM.
<span class="ltx_text ltx_font_bold" id="S4.SS3.p1.1.4">Efficiency wise (x-axis)</span>, <span class="ltx_text ltx_font_smallcaps" id="S4.SS3.p1.1.5">Self-Discover</span> only requires one call per instance and three more inference calls on the task-level, CoT-self-consistency requires 10 times more since we have to sample 10 times for each instance, and methods using each RM requires 40 times more as we use 40 RMs.
In summary, <span class="ltx_text ltx_font_smallcaps" id="S4.SS3.p1.1.6">Self-Discover</span> presents itself a strong reasoning boosting method that is efficient to deploy on large-scale.</p>

<span class="ltx_tag ltx_tag_section">5 </span>Deep Diving Into Self-Discovered Reasoning Structures</h2>
<div class="ltx_para" id="S5.p1">
<p class="ltx_p" id="S5.p1.1">After experimental results showing the effectiveness and efficiency of <span class="ltx_text ltx_font_smallcaps" id="S5.p1.1.1">Self-Discover</span> on a range of reasoning tasks, this section further analyzes <span class="ltx_text ltx_font_bold" id="S5.p1.1.2">are all actions of <span class="ltx_text ltx_font_smallcaps" id="S5.p1.1.2.1">Self-Discover</span> needed</span> and <span class="ltx_text ltx_font_bold" id="S5.p1.1.3">what other benefits can self-discovered structures bring?</span>
In Sec. <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S5.SS1" title="5.1 Importance of Self-Discover Actions ‣ 5 Deep Diving Into Self-Discovered Reasoning Structures ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">5.1</span></a>, we show that it is critical to the model’s performance to use the reasoning structures discovered through the three steps of SELECT, ADAPT and IMPLEMENT.
In Sec. <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S5.SS2" title="5.2 Towards Universality of Discovered Reasoning Structures ‣ 5 Deep Diving Into Self-Discovered Reasoning Structures ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">5.2</span></a>, we demonstrate the <span class="ltx_text ltx_font_bold" id="S5.p1.1.4">universality</span> of the self-discovered reasoning structures by (1) applying the structures discovered by PaLM 2-L to GPT-4, (2) applying the structures discovered by GPT-4 to Llama-2-70B. We further show the commonalities between the reasoning structures and human reasoning patterns in Appendix <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#A5.SS0.SSS0.Px1" title="Model-Discovered Reasoning Structures vs. Human Reasoning Patterns ‣ Appendix E Further Anaysis ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">E</span></a>.</p>
</div>
<section class="ltx_subsection" id="S5.SS1">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">5.1 </span>Importance of <span class="ltx_text ltx_font_smallcaps" id="S5.SS1.1.1">Self-Discover</span> Actions</h3>
<figure class="ltx_figure" id="S5.F8"><img alt="Refer to caption" class="ltx_graphics ltx_centering ltx_img_landscape" height="394" id="S5.F8.g1" src="x8.png" width="789"/>
<figcaption class="ltx_caption ltx_centering"><span class="ltx_tag ltx_tag_figure"><span class="ltx_text" id="S5.F8.3.1.1" style="font-size:90%;">Figure 8</span>: </span><span class="ltx_text" id="S5.F8.4.2" style="font-size:90%;"> <span class="ltx_text ltx_font_bold" id="S5.F8.4.2.1">Ablation study on three <span class="ltx_text ltx_font_smallcaps" id="S5.F8.4.2.1.1">Self-Discover</span> actions on 4 reasoning tasks</span>: all three actions are beneficial for task-solving.</span></figcaption>
</figure>
<div class="ltx_para" id="S5.SS1.p1">
<p class="ltx_p" id="S5.SS1.p1.1">We conduct ablation study on the three actions: SELECT, ADAPT, and IMPLEMENT to analyze the effects of <span class="ltx_text ltx_font_smallcaps" id="S5.SS1.p1.1.1">Self-Discover</span> actions.
Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S5.F8" title="Figure 8 ‣ 5.1 Importance of Self-Discover Actions ‣ 5 Deep Diving Into Self-Discovered Reasoning Structures ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">8</span></a> show results using GPT-4 on 4 reasoning tasks when we apply SELECT (-S) or apply SELECT and ADAPT (-SA) or apply all three actions.
We find that with each stage, model’s zero-shot reasoning capability improve consistently across tasks, indicating that all three actions are beneficial. In particular, after all three actions SAI, the reasoning structures are adapted to be task specific, and bring the most gain to solving the reasoning tasks.</p>
</div>
</section>
<section class="ltx_subsection" id="S5.SS2">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">5.2 </span>Towards Universality of Discovered Reasoning Structures</h3>
<section class="ltx_paragraph" id="S5.SS2.SSS0.Px1">
<h4 class="ltx_title ltx_title_paragraph">Applying PaLM 2-L Discovered Structures to GPT-4</h4>
<figure class="ltx_figure" id="S5.F9"><img alt="Refer to caption" class="ltx_graphics ltx_centering ltx_img_landscape" height="493" id="S5.F9.g1" src="x9.png" width="789"/>
<figcaption class="ltx_caption ltx_centering"><span class="ltx_tag ltx_tag_figure"><span class="ltx_text" id="S5.F9.3.1.1" style="font-size:90%;">Figure 9</span>: </span><span class="ltx_text" id="S5.F9.4.2" style="font-size:90%;"> <span class="ltx_text ltx_font_bold" id="S5.F9.4.2.1">Transferrability tests of optimized prompts (OPRO) and composed structures (<span class="ltx_text ltx_font_smallcaps" id="S5.F9.4.2.1.1">Self-Discover</span>)</span>. The results shown are from GPT-4 using the prompts and structures optimized or composed using PaLM 2-L. We find that self-discovered reasoning structure transfers more robustly than optimized prompts.</span></figcaption>
</figure>
<div class="ltx_para" id="S5.SS2.SSS0.Px1.p1">
<p class="ltx_p" id="S5.SS2.SSS0.Px1.p1.1">We first use a PaLM 2-L model to discover the reasoning structures of 4 reasoning tasks. Then, we apply the resulting reasoning structures to the decoding of GPT-4 as grounding.
We compare our approach to OPRO <cite class="ltx_cite ltx_citemacro_citep">(Yang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib42" title="">2023</a>)</cite> which discovered zero-shot-prompts through optimizations.
We apply OPRO prompts optimized using PaLM 2-L on each task to GPT-4 on the same reasoning tasks. Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#S5.F9" title="Figure 9 ‣ Applying PaLM 2-L Discovered Structures to GPT-4 ‣ 5.2 Towards Universality of Discovered Reasoning Structures ‣ 5 Deep Diving Into Self-Discovered Reasoning Structures ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">9</span></a> shows that <span class="ltx_text ltx_font_smallcaps" id="S5.SS2.SSS0.Px1.p1.1.1">Self-Discover</span> outperforms OPRO on 3 out of 4 tasks despite that OPRO used 20% data to optimize the prompt.
In contrast, <span class="ltx_text ltx_font_smallcaps" id="S5.SS2.SSS0.Px1.p1.1.2">Self-Discover</span> is done in a zero-shot manner, demonstrating the efficiency of our method and universality of the discovered reasoning structures.
</p>
</div>
</section>
<section class="ltx_paragraph" id="S5.SS2.SSS0.Px2">
<h4 class="ltx_title ltx_title_paragraph">Applying GPT-4 Discovered Structures to Llama2 and ChatGPT</h4>
<div class="ltx_para" id="S5.SS2.SSS0.Px2.p1">
<p class="ltx_p" id="S5.SS2.SSS0.Px2.p1.1">Motivated by transferrability performance across LLMs, we further investigate can self-discovered reasoning structures from LLMs boost reasoning for <span class="ltx_text ltx_font_italic" id="S5.SS2.SSS0.Px2.p1.1.1">smaller LMs</span> that are challenging to come up with structures themselves<span class="ltx_note ltx_role_footnote" id="footnote3"><sup class="ltx_note_mark">3</sup><span class="ltx_note_outer"><span class="ltx_note_content"><sup class="ltx_note_mark">3</sup><span class="ltx_tag ltx_tag_note">3</span>We tried zero-shot meta prompting Llama2 but observed low-quality structure outputs.</span></span></span>.
We use GPT-4 to discover the task-intrinsic reasoning structures, and then apply those structures to the decoding of open-sourced Llama2-70B as well as GPT-3.5-turbo (ChatGPT) on two subsets of tasks from BBH.
We find that using self-discovered structures on Llama2 (52%) outperforms CoT (42%) on disambiguation QA zero-shot and on GPT-3.5-turbo (56%) outperforms CoT (51%) on geometry with 3-shot demonstration from structured reasoning process.</p>
</div>

<section class="ltx_section" id="S6">
<h2 class="ltx_title ltx_title_section">
<span class="ltx_tag ltx_tag_section">6 </span>Related Work</h2>
<section class="ltx_subsection" id="S6.SS1">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">6.1 </span>Prompting Methods</h3>
<div class="ltx_para" id="S6.SS1.p1">
<p class="ltx_p" id="S6.SS1.p1.1">Recent advancements in the area of LLMs have given rise to a plethora of few-shot <cite class="ltx_cite ltx_citemacro_citep">(Brown et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib3" title="">2020</a>)</cite> and instruction <cite class="ltx_cite ltx_citemacro_citep">(Mishra et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib23" title="">2022c</a>; Wei et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib40" title="">2021</a>; Ouyang et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib29" title="">2022</a>)</cite> prompting techniques, including <span class="ltx_text ltx_font_italic" id="S6.SS1.p1.1.1">Chain-of-Thought</span> prompting (CoT) <cite class="ltx_cite ltx_citemacro_citep">(Nye et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib25" title="">2021</a>; Wei et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib41" title="">2022</a>)</cite>, Least-to-most prompting <cite class="ltx_cite ltx_citemacro_citep">(Zhou et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib48" title="">2022a</a>; Drozdov et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib9" title="">2022</a>)</cite>, Decomposed prompting <cite class="ltx_cite ltx_citemacro_citep">(Khot et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib16" title="">2022</a>)</cite>, Reframing <cite class="ltx_cite ltx_citemacro_citep">(Mishra et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib22" title="">2022b</a>)</cite>, Help Me Think Prompting <cite class="ltx_cite ltx_citemacro_citep">(Mishra &amp; Nouri, <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib20" title="">2023</a>)</cite>, Stepback Prompting <cite class="ltx_cite ltx_citemacro_citep">(Zheng et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib46" title="">2023</a>)</cite> and search-based approaches like <span class="ltx_text ltx_font_italic" id="S6.SS1.p1.1.2">Tree-of-Thought (ToT)</span> <cite class="ltx_cite ltx_citemacro_citep">(Yao et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib43" title="">2023a</a>)</cite>, Graph-of-Thought <cite class="ltx_cite ltx_citemacro_citep">(Besta et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib2" title="">2023</a>; Yao et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib44" title="">2023b</a>)</cite>, Branch-solve-merge <cite class="ltx_cite ltx_citemacro_citep">(Saha et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib33" title="">2023</a>)</cite> and RAP <cite class="ltx_cite ltx_citemacro_citep">(Hao et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib13" title="">2023</a>)</cite>. Each of the prompting methods has some strengths and weaknesses in terms of their successful application domain. Our work <span class="ltx_text ltx_font_smallcaps" id="S6.SS1.p1.1.3">Self-Discover</span> presents the missing piece in the prompting literature, as <span class="ltx_text ltx_font_smallcaps" id="S6.SS1.p1.1.4">Self-Discover</span> provides a way to self-compose over various prompting methods via the proposed self-discovery mechanism. Composing over prompting methods in <span class="ltx_text ltx_font_smallcaps" id="S6.SS1.p1.1.5">Self-Discover</span> is analogous to the programming literature where a program is written using various basic building blocks such as for loop, if/else condition etc.</p>
</div>
</section>
<section class="ltx_subsection" id="S6.SS2">
<h3 class="ltx_title ltx_title_subsection">
<span class="ltx_tag ltx_tag_subsection">6.2 </span>Reasoning and Planning</h3>
<div class="ltx_para" id="S6.SS2.p1">
<p class="ltx_p" id="S6.SS2.p1.1">With the development of various reasoning and planning benchmarks such as GSM8K <cite class="ltx_cite ltx_citemacro_citep">(Cobbe et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib8" title="">2021</a>)</cite>, Math <cite class="ltx_cite ltx_citemacro_citep">(<a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib14" title="">Hendrycks et al., </a>)</cite>, BigBench <cite class="ltx_cite ltx_citemacro_citep">(Srivastava et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib34" title="">2023</a>)</cite> etc., various methods have been proposed to improve model performance. Often these methods induce specific reasoning structures mimicking the reasoning structure of the underlying task associated with the dataset. For example, chain of thought <cite class="ltx_cite ltx_citemacro_citep">(Wei et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib41" title="">2022</a>)</cite> and scratchpad <cite class="ltx_cite ltx_citemacro_citep">(Nye et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib25" title="">2021</a>)</cite> induce generation of explanations associated with a reasoning question. Similarly other methods induces specific reasoning structures such as question summarization <cite class="ltx_cite ltx_citemacro_citep">(Kuznia et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib18" title="">2022</a>)</cite>, question decomposition <cite class="ltx_cite ltx_citemacro_citep">(Patel et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib30" title="">2022</a>)</cite>, program generation <cite class="ltx_cite ltx_citemacro_citep">(Mishra et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib21" title="">2022a</a>; Chen et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib5" title="">2022</a>; Gao et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib12" title="">2023b</a>)</cite>, etc. However, in a real world user traffic, queries can be diverse covering various reasoning structures. Our work <span class="ltx_text ltx_font_smallcaps" id="S6.SS2.p1.1.1">Self-Discover</span> allows models to combine multiple reasoning approaches by self-composing into a structure without the need to access task labels. There have been some related work that explores LLM combining skills in-context such as SkiC <cite class="ltx_cite ltx_citemacro_citep">(Chen et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib4" title="">2023</a>)</cite>, devising a strategy <cite class="ltx_cite ltx_citemacro_citep">(Gao et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib11" title="">2023a</a>)</cite>, and planning with iterative quering <cite class="ltx_cite ltx_citemacro_citep">(Liu et al., <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#bib.bib19" title="">2023</a>)</cite>.
However, they require human annotating skills and reasoning plans while <span class="ltx_text ltx_font_smallcaps" id="S6.SS2.p1.1.2">Self-Discover</span> leverages a scalable solution with the help of LLM’s meta-task reasoning capabilities.</p>
</div>
</section>
</section>
<section class="ltx_section" id="S7">
<h2 class="ltx_title ltx_title_section">
<span class="ltx_tag ltx_tag_section">7 </span>Conclusion</h2>
<div class="ltx_para" id="S7.p1">
<p class="ltx_p" id="S7.p1.1">We introduce <span class="ltx_text ltx_font_smallcaps" id="S7.p1.1.1">Self-Discover</span>, an efficient and performant framework for models to self-discover a reasoning structure for any task from a seed set of general problem-solving skills.
We observe drastic improvements on challenging reasoning benchmarks from multiple LLMs up to 30%.
Ablations study of <span class="ltx_text ltx_font_smallcaps" id="S7.p1.1.2">Self-Discover</span> demonstrates that the composed reasoning structures are universally transferable between LLMs.
Forward looking, we are excited to explore more on LLM structured reasoning to push the boundary of problem-solving and discover potentials for Human-AI collaboration.</p>
</div>

<section class="ltx_appendix" id="A2">
<h2 class="ltx_title ltx_title_appendix">
<span class="ltx_tag ltx_tag_appendix">Appendix B </span>Evaluation Details</h2>
<div class="ltx_para" id="A2.p1">
<p class="ltx_p" id="A2.p1.1">We use accuracy and exact matching as with other methods tested on BBH, T4D and MATH.
To properly evaluate the generated answers from LLMs, we prompt the models to end the answer with “<span class="ltx_text ltx_font_italic" id="A2.p1.1.1">Thus, the final answer is [X]</span>”, where X is either one answer option such as “<span class="ltx_text ltx_font_italic" id="A2.p1.1.2">A</span>” or a string such as “<span class="ltx_text ltx_font_italic" id="A2.p1.1.3">valid</span>”.
During evaluation, we manually examine each task’s outputs from LLMs and design heuristics to extract the final answers. For MATH dataset, we find that it is challenging to extract the answers accurately. As a result, we subsample 200 test examples from MATH, and manually sanity check and annotate the extracted answers for all methods tested in our paper.</p>
</div>

<div class="ltx_para" id="A4.p1">
<p class="ltx_p" id="A4.p1.1">We perform an error analysis of <span class="ltx_text ltx_font_smallcaps" id="A4.p1.1.1">Self-Discover</span> on the MATH dataset of 200 samples to understand the failure modes. <div class="ltx_para" id="A4.p3">
<p class="ltx_p" id="A4.p3.1">Next, we analyze the errors made by the model in <span class="ltx_text ltx_font_smallcaps" id="A4.p3.1.1">Self-Discover</span>: out of 99 examples where the model prediction is wrong, wrong reasoning structures account for only 25.3% of the errors. The remaining 74.7% errors are due to errors in the intermediate calculations such as math computations. Table <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#A4.T5" title="Table 5 ‣ Appendix D Error Analysis ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">5</span></a> shows 3 examples of such errors. This insight indicates that future improvements should aim at improving the step-wise calculation accuracy of LLMs, such as using tools or code generation.</p>

<section class="ltx_paragraph" id="A5.SS0.SSS0.Px1">
<h4 class="ltx_title ltx_title_paragraph">Model-Discovered Reasoning Structures vs. Human Reasoning Patterns</h4>
<div class="ltx_para" id="A5.SS0.SSS0.Px1.p1">
<p class="ltx_p" id="A5.SS0.SSS0.Px1.p1.1">We investigate whether LLM-discovered reasoning structures share some commonalities with human reasoning patterns.
We give humans 3 task instances without labels and an example reasoning structure (same as <span class="ltx_text ltx_font_smallcaps" id="A5.SS0.SSS0.Px1.p1.1.1">Self-Discover</span> meta-reasoning stage) and ask them to write a reasoning structure for a task before solving it.
Figure <a class="ltx_ref" href="https://arxiv.org/html/2402.03620v1#A5.F11" title="Figure 11 ‣ Appendix E Further Anaysis ‣ Self-Discover: Large Language Models Self-Compose Reasoning Structures"><span class="ltx_text ltx_ref_tag">11</span></a> shows comparison of human and LLM-composed reasoning structures on the BBH-navigation task.
We observe similar structures such as mental-noting after each movement.
From promising findings of LLM self-discovered structures boost and share traits of human meta-reasoning, we hope to encourage more future work to study humna-AI collaboration for complex problem-solving.
</p>
